# Cross-Branch Sync

各并行分支 / agent 之间的异步协调目录。

## 机制

- 每个分支在此目录下维护自己的文件（如 `urban.md`、`master.md`）
- 有需要通知其他分支的事项时，写入自己的文件并提交
- 其他分支定期或按需阅读，按状态标记处理结果
- 无固定分支数量，按需创建文件即可

## 记录格式

每条记录包含：

```
### [状态] 标题
- **发出方**：branch-name
- **接收方**：all / branch-name（多个用逗号分隔）
- **日期**：YYYY-MM-DD
- **内容**：具体说明
- **是否需要处理**：是 / 否（仅通知）
- **处理方式**：（需要处理时填写）
```

状态标记：
- `[待处理]` — 接收方尚未处理
- `[仅通知]` — 无需处理，知悉即可
- `[已处理]` — 接收方已完成（由接收方更新）
