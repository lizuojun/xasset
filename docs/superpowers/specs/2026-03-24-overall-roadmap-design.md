# JiaJia 整体产品规划

**日期：** 2026-03-24
**状态：** 已确认
**架构方案：** 方案 C（双层资产模型）

---

## 产品定位

面向 **IP/内容创作者** 的 AI 驱动3D场景资产生成与管理平台。

- **输入：** 照片、概念原画、文字描述（多模态）
- **输出：** USD/GLTF 格式资产，供 Omniverse 等平台使用
- **核心价值：** 可拆分、可重组的高质量3D场景资产库
- **商业化：** 创作者资产市场，平台抽成，支持整体或部分资产交易

---

## 架构核心：双层资产模型

```
原始图像 / 文字描述
        │
        ▼
  ┌─────────────┐
  │  Raw Asset  │  ← AI Pipeline 操作层
  │  (JSON +    │     可迭代加工、组合、拆分
  │   几何数据)  │
  └──────┬──────┘
         │ 发布 / 打包
         ▼
  ┌──────────────────┐
  │ Packaged Asset   │  ← 交付与商业化层
  │ (USD + 元数据    │     版本记录、水印、版权链
  │  + 水印 + 版权)  │
  └──────────────────┘
```

---

## AI 生成 Pipeline（三阶段）

| 阶段 | 内容 |
|------|------|
| **Stage 1** | 场景理解与基础地理生成：场景类型识别、场景划分、地形/植被/水系 Mesh 生成 |
| **Stage 2** | 模型生成/匹配 + 布局组合：单体模型生成或从库中匹配，按布局逻辑组合 |
| **Stage 3** | 风格化处理 + 光照生成 |
| **附加能力** | 风格迁移、寻路生成、碰撞体生成 |

## 子项目规划

### Phase 0：基础定义

| 编号 | 子项目 | 核心内容 | 依赖 |
|------|--------|----------|------|
| P0 | **资产数据模型 & Schema 设计** | Raw Asset Schema、Packaged Asset Schema、商业化元数据结构、资产层次定义（urban/wild/house 体系） | 无 |

### Phase 1：核心能力

| 编号 | 子项目 | 核心内容 | 依赖 |
|------|--------|----------|------|
| P1 | **AI 生成 Pipeline** | 三阶段生成链路、多模型编排、输入适配（图像/文字/原画） | P0 |
| P2 | **基础设施** | OSS 存储、任务队列、数据库、REST/GraphQL API 层 | P0 |

> P1 与 P2 可并行推进

### Phase 2：产品化

| 编号 | 子项目 | 核心内容 | 依赖 |
|------|--------|----------|------|
| P3 | **资产库 Web 应用** | 上传、浏览、标注、拆分组合、导出、资产管理 | P0、P2 |
| P4 | **商业化系统** | 版本记录、水印机制、资产市场、交易流程、平台抽成 | P3 |

### Phase 3：生态连接

| 编号 | 子项目 | 核心内容 | 依赖 |
|------|--------|----------|------|
| P5 | **Omniverse 插件** | 触发生成、预览、USD 资产导入 | P0、P2 |
